# Lv.1 個人の作業効率化 — AIとの協働ガイドライン

## このページについて

Lv.1では、エンジニア個人がAIをツールとして使いこなし、日々の作業スピードと品質を上げることを目指します。

各ガイドラインページに書かれた協働フローを分析すると、どの作業においても共通する原則が浮かび上がります。
本ページはその原則をまとめたものです。**新しい作業でAI活用を検討するとき、このガイドラインを指針として活用してください。**

---

## 協働の基本原則

> **「人が文脈を渡し、AIが構造化する。人が判断し、AIが展開する。」**

AIは万能ではありませんが、特定の作業において人間をはるかに上回ります。
Lv.1では、この得意・不得意の違いを理解した上で、役割を意図的に分担することが重要です。

---

## 人とAIの役割分担

### 人が担う役割

| 役割 | 具体的な行動 |
| :--- | :--- |
| **文脈の提供** | 業務ルール・要件・制約をAIに渡せる形で整理する |
| **業務判断** | 生成された成果物が業務要件と合っているかを判断する |
| **レビューと修正** | AIの出力の方向性を正し、精緻化を依頼する |
| **最終承認** | 成果物を確定させて次工程に引き渡す |

### AIが担う役割

| 役割 | 具体的な行動 |
| :--- | :--- |
| **構造化** | 散らかったテキストを表・マトリクス・設計書に整理する |
| **可視化** | 要件をフロー図・遷移図・シーケンス図に変換する |
| **網羅性の担保** | 人が見落としがちなパターン・ケース・制約を洗い出す |
| **展開・生成** | 確定した構造から大量のドラフト・コード・テストを一気に生成する |

---

## 協働フローの共通構造

Lv.1の全ガイドラインページは、以下の共通サイクルで設計されています。

```
1. 👤 人：インプットを整える
        ↓
2. 🤖 AI：ドラフトを素早く生成する
        ↓
3. 👤 人：レビューして方向を修正する
        ↓
4. 🤖 AI：精緻化・次の成果物を生成する
        ↓
5. 👤 人：最終確認・承認して次工程へ渡す
```

このサイクルは1回とは限りません。ステップ3→4を複数回繰り返しながら、成果物の質を高めていきます。

## まだガイドラインがない作業への適用方法

各ガイドラインページに載っていない作業でも、以下の手順でAI活用を設計できます。

### Step 1: 作業の「本質的な目的」を考える

ドキュメントの名前や作業の名称ではなく、**「この作業は何のために存在するのか」** を問います。

- 誰かとの認識を揃えるため？
- 後工程の手戻りを防ぐため？
- 実装の迷いをなくすため？

### Step 2: AIに渡せる「インプット」を特定する

AIが動くには、構造化の素材となるテキストが必要です。

- 既存のドキュメント（要件定義書・仕様書・ER図など）
- 口頭で共有されている業務ルール
- 箇条書きのメモや過去の成果物

### Step 3: AIに任せる作業と、人が判断する作業を分ける

| AIに任せてよい作業 | 人が担うべき作業 |
| :--- | :--- |
| テキストから表・図・一覧の生成 | 業務ルールとの整合性チェック |
| パターン・ケースの網羅的な洗い出し | 優先順位・方針の決定 |
| 確定した構造からのコード・テスト生成 | 最終的な正しさの判断 |
| ドラフトの素早い作成 | 成果物の承認と次工程への連携 |

### Step 4: 「ドラフト生成 → レビュー」のサイクルで進める

完璧なアウトプットを一発で求めるのではなく、まずAIに粗いドラフトを出させ、そこから対話で精緻化する姿勢が重要です。
