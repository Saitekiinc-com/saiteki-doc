# Level 2: シェア型 (Shared / Collaborative)

## コンセプト
**「新しい開発アプローチと合意形成の加速」**

AIをチーム内での「合意形成のための触媒」として活用する段階です。AI生成によって従来の手法を変革し、ステークホルダー間で「実際に動くもの」を共有しながら開発を進めるスタイルです。

## 具体的な取り組み

### 1. HTMLモック早期生成 (Prototyping)
- **内容**: 要件定義〜詳細設計のフェーズにおいて、仕様書を書く前にAIを用いてHTML/CSSで動くモックを作成する。
- **効果**:
    - 言葉だけでは伝わりにくいUIの挙動やデザインを早期に可視化できる。
    - デザイナーやプロダクトオーナー（顧客）と、実際の画面を見ながら認識合わせができるため、手戻りが激減する。

### 2. 合意形成の早期化 (Consensus Building)
- **内容**: 従来のウォーターフォール的な「要件定義書 → 設計書 → 実装」というフローではなく、「モック作成 → 合意 → 実装」というAIネイティブなフローへ移行する。
- **効果**: エンジニア側へのインプットが明確になり、実装フェーズでの迷いがなくなる。開発スピードと顧客満足度が同時に向上する。
