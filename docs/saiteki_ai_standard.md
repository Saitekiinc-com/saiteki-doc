# Saiteki AI Standard：課題解決と未来への適応

## 1. Vision：Saitekiが目指す組織とエンジニア像

AI導入の先に目指すのは、単なる効率化ではありません。Saitekiのエンジニアと組織が以下のような状態になることを目指します。

### 組織の理想像 (Organizational Ideal)
*   **「単純作業ゼロ」の組織**:
    *   定型的なコーディング、テスト記述、ドキュメント更新はAIとシステムが自律的に処理し、人間は「意思決定」と「創造」に100%のリソースを割いている。
*   **「集合知」が即座に使える組織**:
    *   組織内の全ドキュメント、コード、過去の議論がAIによって構造化され、誰でも瞬時に必要な情報にアクセス・活用できる。

### エンジニアの理想像 (Employee Ideal)
*   **AI Orchestrator (AIの指揮者)**:
    *   自らコードを書くだけでなく、AIに適切な指示（プロンプト/コンテキスト）を与え、複数のAIエージェントを指揮して、一人でチーム並みのアウトプットを出す。
*   **Value Creator (価値の創造者)**:
    *   「どう作るか（How）」の悩みから解放され、「何を作るか（What）」「なぜ作るか（Why）」という顧客価値の本質に向き合っている。

## 2. AI駆動開発の成熟度モデル (AI-Driven Development Levels)

| Level | フェーズ名 | 状態定義 | 具体的な業務イメージ |
| :--- | :--- | :--- | :--- |
| **Lv.1** | **Individual** | **個人支援** | **「個人の作業効率化」**<br>コード補完、調査支援、要件定義ドラフト作成など、個人の手元作業をAIで高速化する。 |
| **Lv.2** | **Shared** | **シェア型** | **「合意形成の加速」**<br>HTMLモックやAPIスキーマの早期生成により「動くもの・構造」を素早く共有し、チーム全体の合意形成や並行開発を実現する。 |
| **Lv.3** | **Autonomous** | **自律型** | **「組織能力の拡張」**<br>オンボーディング計画生成、日報ナレッジ蓄積、エキスパートマッチングなど、AIエージェントが自律的に組織の課題を解決する。 |
