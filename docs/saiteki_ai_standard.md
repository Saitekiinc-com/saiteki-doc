# Saiteki AI Standard

本ドキュメントは、SaitekiにおけるAI活用の指針と、それを実現するためのロードマップを定義したものです。

## このドキュメントについて

本ドキュメントは、**仮説検証を繰り返しながら育てていく生きた資料**です。

「AIにこの作業を任せられるはず」という仮説を記述し、実際の案件を通じて検証し、結果を追記する。このサイクルを回し続けることで、Saitekiの現場で通用する**確度の高いAI活用ナレッジ**を積み上げていきます。

初期の段階から完璧な答えを出そうとするのではなく、**小さな仮説を実践で磨き続けること**をこのドキュメントの根本方針とします。案件での気づき・フィードバック・修正が蓄積されるほど、ここに書かれている内容の信頼性は高まります。

## 目指す姿

### 組織の理想像：集合知が活きる組織

Saitekiが目指すのは、**個人の頭の中にあるナレッジを組織の資産に変える**ことができる組織です。

- **集合知の活用**: 誰かが案件で得た気づき・工夫・判断基準が、ドキュメントとして蓄積され、他のメンバーが同じ課題にぶつかったときに即座に参照できる。
- **取りこぼしをなくす仕組み**: 「あの人なら知っていた」「前にやったことがあったのに」という機会損失が、AIとナレッジグラフの連携によって防がれている。
- **経験が資産になる組織**: 案件を重ねるほど組織が賢くなり、後から入ったメンバーも先人の知見をすぐに活用できる。

### エンジニアの理想像：価値を生み出すことを考える

SaitekiのエンジニアがAIと協働することで目指す姿は、**「どう作るか」ではなく「何を作るか・なぜ作るか」を考える時間を増やすこと**です。

- **AIに任せられる作業を見極める力**: 定型的な調査・実装・ドキュメント化をAIに委ねながら、自分の判断が本当に必要な場所に集中できる。
- **フィードバックを資産に変える習慣**: 案件で「うまくいった」「うまくいかなかった」を言語化して蓄積し、チーム全体のAI活用精度を高めることに貢献する。
- **価値創出への集中**: 実装速度という制約から解放され、顧客への提供価値・設計の妥当性・技術的な意思決定に思考を集中できる。

## AI駆動開発の成熟度モデル

Saitekiでは、AI活用の深度を以下の3段階で定義しています。現在地を把握し、次のステップへ進むための指針として活用してください。

### Lv.1 — Individual（個人支援）

**「個人の作業効率化」**

エンジニア個人がAIを手元のツールとして活用し、日々の作業スピードを上げる段階です。

- コード補完・調査・テストケース作成・ドキュメントドラフトなどを個人レベルでAI化する
- 「使ってみた」から「使いこなす」習慣へ移行することが目標
- このドキュメントのガイドライン（Lv.1）では、テスト観点抽出・テスト戦略・テスト設計書作成などの実践フローを提供

### Lv.2 — Shared（シェア型）

**「作業領域の拡大」**

個人の活用をチームに広げ、AIが担う作業の範囲をチーム全体へ拡張する段階です。AIが生成したアウトプットを早期に共有することで、並行開発を加速し、手戻りを削減します。

- HTMLモック・APIスキーマ・データモデルを早期に生成し、「動くもの・構造」でチームに共有する
- 設計の手戻りや認識のズレを、早いフィードバックサイクルで防ぐ
- AIが担える作業領域を個人単位から、チーム全体へ広げることが目標

### Lv.3 — Autonomous（自律型）

**「組織能力の拡張」**

AIとナレッジグラフがシステムとして連携し、個人やチームを超えて組織レベルでAIが機能する段階です。

- issueが作成されるとAIが自動でナレッジを検索し、AI活用ガイダンスのコメントを届ける
- 案件を重ねるたびにナレッジが蓄積・精緻化され、次のタスクのAIコメントがより精度高くなる
- 組織の集合知をAIが仲介することで、属人化を解消し、チーム全体の判断品質を底上げする

## サイクルのイメージ

```
案件・タスク
    ↓
「AIに任せられそう」という仮説をNotionに記述
    ↓
GitHub Pagesに公開 → ナレッジグラフに蓄積
    ↓
issueでAIコメントを受け取り、タスクを進める
    ↓
「実際どうだったか」の気づきをNotionに追記
    ↓
ナレッジが更新され、次のタスクの精度が上がる
    ↓
（繰り返し）
```

この仮説検証のサイクルを積み重ねることが、Saitekiのエンジニアと組織を強くする唯一の方法です。
