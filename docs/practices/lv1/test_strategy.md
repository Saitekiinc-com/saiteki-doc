# テスト戦略の策定

## 【背景】なぜこのフローが必要なのか

テスト戦略の策定は、「何を・どこまで・どのように」テストするかの方針を決める工程です。
この判断が曖昧なまま進むと、重要なリスクが見落とされたり、逆に低リスクな箇所に工数が集中するといった非効率が生まれます。

### AI活用による課題解決のアプローチ

本カリキュラムでは、**「人の判断 × AIの情報処理」** を組み合わせることで、品質リスクを構造的に把握し、合理的な戦略を高速に策定します。

1.  **情報過多の解消**: 仕様書・設計書・過去障害データをAIが解析・整形し、戦略判断に必要なエッセンスを抽出する。
2.  **リスクの可視化**: 品質リスクとその発生確率・影響度をAIが整理し、人が優先度を判断するための材料を提供する。
3.  **戦略文書の高速化**: テスト方針・体制設計のドラフトをAIが生成し、人は最終判断と調整に集中する。

## フローの概要

プロジェクトの制約条件の中で、品質リスクを優先的に潰すためのテスト方針・体制を策定するフローです。

| AIへのインプット | AIからアウトプット |
| :--- | :--- |
| 要件定義<br>画面仕様書<br>画面一覧・遷移図<br>機能一覧表<br>過去プロジェクトデータ | テスト戦略<br>品質リスク一覧<br>テスト体制設計書 |

### 協働フロー

1. 👤 **ユーザー (Human)** + 🤖 **生成AI (AI)** — **情報構造化**
    *   人が各資料の役割を定義し、戦略判断に不要な情報（UI文言・実装コードなど）を除去します。
    *   AIへの投入単位をブロックごとに設計します（システム概要 / 機能分類 / 画面遷移 / 過去障害）。
    *   AIが情報を整形し、システム特性と「リスクの種」を抽出します。

2. 👤 **ユーザー (Human)** + 🤖 **生成AI (AI)** — **システム特性抽出**
    *   人が重要機能と制約条件（納期・人員・技術負債）を提示します。
    *   AIがリスク構造を可視化し、壊れやすいポイントを推定します。

3. 👤 **ユーザー (Human)** + 🤖 **生成AI (AI)** — **品質リスク抽出**
    *   人がリスクの評価基準（影響度・発生確率・発見困難性）を提示します。
    *   AIが機能リスク・非機能リスクを両面から抽出し、根拠とともに候補一覧を生成します。

4. 👤 **ユーザー (Human)** — **リスク優先度決定**
    *   AIが生成したリスク候補一覧をもとに、影響度・確率を評価します。
    *   経営・事業視点で最終的な優先順位を決定します。
    *   *目的: 優先度の判断は必ず人が行う。AIは候補を出す役割*

5. 🤖 **生成AI (AI)** + 👤 **ユーザー (Human)** — **テスト方針決定（戦略生成）**
    *   AIが各リスクに対応するテストアプローチ（自動化・探索的テスト・負荷テストなど）を提案します。
    *   AIがテスト戦略文書のドラフトを生成します。
    *   人がやらない範囲を明確化し、方針を確定させます。

6. 🤖 **生成AI (AI)** + 👤 **ユーザー (Human)** — **テスト体制設計**
    *   人が組織制約・スキルマップ・役割定義を提示します。
    *   AIが体制案・リソース配分・スケジュール概算を生成します。
    *   人が現実的な制約に合わせて最終調整を行います。
