# ユーザー操作とフロントロジックの接続設計をAIで効率化する

## 【背景】なぜこのフローが必要なのか

「ボタンを押したら何が起きるか」「入力値が不正なら何を表示するか」——こうしたユーザー操作に対するシステムの振る舞いは、実装直前に初めて詳細を詰めることが多く、その結果として実装・QA工程で認識のズレが発覚することがあります。

AIを活用することで、画面定義と要件からバリデーションルール・操作フロー・処理の流れをドラフト生成できます。
**「実装前に操作の振る舞いを全量定義する」** ことで手戻りとバグを防ぐことが目的です。

### AI活用による課題解決のアプローチ

1. **バリデーション定義の網羅**: 入力項目ごとのバリデーションルールをAIが洗い出し、フロント・バック双方で統一できる定義書を生成します。
2. **操作フローの可視化**: ユーザーの操作（クリック・入力・遷移）に対するシステムの応答をAIが整理し、抜け漏れを防ぎます。
3. **フロントロジックの事前整理**: 処理フローをAIと協働で設計することで、実装時の判断コストを下げます。

## フローの概要

画面設計と機能要件をインプットに、操作・バリデーション・フロントロジックの設計ドキュメントを作成するフローです。

| AIへのInput | AIからのOutput |
| :--- | :--- |
| **画面の入力項目一覧・業務ルール** | **入力項目・バリデーション定義**（条件・エラーメッセージ含む） |
| **画面一覧・ユーザーストーリー** | **イベント・操作と遷移先マトリクス**（操作→結果のマッピング） |
| **機能要件・イベント定義** | **フロント側処理フロー**（APIコール・状態更新のシーケンス） |

### 協働フロー

1. 👤 **ユーザー** - **入力項目と業務ルールの整理**
    - 各画面の入力項目と、それに紐づく業務ルール（必須・文字数・形式等）をテキストで整理します。

2. 🤖 **生成AI** - **バリデーション定義の生成**
    - 入力項目と業務ルールをもとに、各項目のバリデーションルール・エラーメッセージ・表示タイミングを整理した一覧を生成します。
    - *目的: フロント・バック双方で同じルールを実装するためのドキュメントを作るため*

3. 👤 **ユーザー** - **ルールの確認と修正**
    - 生成されたバリデーション定義を確認し、業務ルールと照合して修正します。

4. 🤖 **生成AI** - **操作・遷移マトリクスの生成**
    - 各画面のユーザー操作（ボタン押下・フォーム送信・リンクタップ等）と、それに対するシステムの応答・遷移先をマトリクス形式で生成します。
    - *目的: 操作パターンの抜け漏れを防ぎ、QAのテスト観点と一致させるため*

5. 🤖 **生成AI** - **フロント側処理フローの生成**
    - 確定した操作定義をもとに、APIコール・状態更新・エラーハンドリングの流れをシーケンス図またはMarkdown形式で生成します。
    - *目的: 実装者が処理の流れを把握して迷わず実装できるようにするため*

6. 👤 **ユーザー** - **フローの最終確認**
    - 処理フローの整合性を確認し、バックエンドの仕様と矛盾がないかを検証します。
