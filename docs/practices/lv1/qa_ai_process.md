# 高品質なテストを実現するQAプロセスのAI活用

## 【背景】なぜこのフローが必要なのか

テスト工程は開発の最終局面で行われることが多く、スケジュールの圧迫を受けやすい領域です。また、テスト観点の抜け漏れや場当たり的なケース作成は、リリース後の重大なバグを招くリスクを孕んでいます。
従来のQAプロセスでは、経験豊富なテスターの知見に頼る部分が大きく、属人化と網羅性の担保が課題となっていました。

### AI活用による課題解決のアプローチ
本ガイドラインでは、特に効果が高い作業項目を中心にAIを活用し、品質の底上げと工数削減を両立させます。

1.  **網羅性の担保**: AIが仕様から多角的なテスト観点を自動抽出することで、人間の盲点をカバーします。
2.  **リスクの早期可視化**: テスト戦略段階からAIとリスク分析を行うことで、手戻りを最小限に抑えます。
3.  **作業の高速化**: 構造が決まっているテストケースの実装やデータ準備をAIに任せ、人間は「判断と最終確認」に注力します。

## フローの概要

最優先でAIを活用すべき「星5」の作業項目（観点抽出・戦略・計画・設計・実装）を統合したフローです。

| AIへのInput | AIからのOutput |
| :--- | :--- |
| **要件定義書・基本設計書** | **テスト観点一覧**（網羅性チェック用） |
| **プロジェクト特性・納期・制約** | **テスト戦略案・優先順位**（松竹梅プラン） |
| **テスト計画の骨子** | **テスト計画書**（ドラフト） |
| **テスト観点・機能一覧** | **テスト設計書** |
| **テスト設計書・指定フォーマット** | **実行用テストケース**（手順・データ付） |

### 協働フロー

1. 🤖 **生成AI (AI)** - **テスト観点の抽出**
    *   要件定義書等のドキュメントを読み込み、正常系・異常系・境界値などの多角的なテスト観点を抽出します。
    *   *目的: テスト漏れを防ぎ、網羅性を向上させるため*

2. 👤 **ユーザー (Human)** - **テスト戦略の確定**
    *   AIが提示したリスク分析案をもとに、今回のリリースの最優先事項を決定し、テスト範囲（自動化対象/手動対象など）を確定させます。

3. 🤖 **生成AI (AI)** - **テスト設計とテストケースの実装**
    *   確定した戦略と設計方針に基づき、実際の手順や期待値を含む詳細なテストケース（およびテストデータ）を自動生成します。
    *   *目的: 単調な入力・展開作業を自動化し、レビュー効率を上げるため*

4. 👤 **ユーザー (Human)** - **最終レビューと承認**
    *   AIが生成した成果物に対し、「業務要件との整合性」や「現実的な実行可能性」の観点で最終チェックを行います。
