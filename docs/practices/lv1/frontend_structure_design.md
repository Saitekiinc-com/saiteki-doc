# フロントエンドの構造分割設計をAIで効率化する

## 【背景】なぜこのフローが必要なのか

詳細設計フェーズで「どの画面に何のコンポーネントを置くか」「どうコンポーネントを分割するか」を事前に設計しないまま実装を始めると、コードが画面ごとに重複し、後から修正するたびに複数箇所を直す必要が生じます。

AIを活用することで、画面一覧・要件テキストから構造設計のドラフトを素早く生成できます。
**「実装着手前に、分割の境界線を引く」** 設計をAIとの協働で個人レベルで実現することが目的です。

### AI活用による課題解決のアプローチ

1. **コンポーネント設計の高速化**: 画面一覧と要件テキストから、再利用可能なコンポーネントの候補をAIが洗い出します。
2. **分割基準の統一**: AIが「単一責任の原則」に基づいてコンポーネントの責務を整理し、設計の一貫性を保ちます。
3. **実装分担の明確化**: コンポーネントの一覧と責務定義が揃うことで、複数人での実装分担がしやすくなります。

## フローの概要

画面一覧と要件定義をインプットに、コンポーネント設計ドキュメントを作成するフローです。

| AIへのInput | AIからのOutput |
| :--- | :--- |
| **画面一覧・ワイヤーフレーム** | **各画面の構成要素一覧**（画面を構成するUI要素の洗い出し） |
| **構成要素一覧・デザイン方針** | **コンポーネント分割案**（親子関係・再利用性を含む） |
| **コンポーネント分割案** | **コンポーネント設計書**（責務・props・表示状態の定義） |

### 協働フロー

1. 👤 **ユーザー** - **画面一覧の確認**
    - 基本設計で作成した画面一覧・ワイヤーフレームをインプットとして準備します。
    - *ポイント: 画面IDが付いていると、AIとのやりとりがスムーズになります*

2. 🤖 **生成AI** - **各画面の構成要素の洗い出し**
    - 各画面に含まれるUI要素（ヘッダー・フォーム・リスト・ボタン等）を洗い出し、構成要素一覧を生成します。
    - *目的: 「何を作るか」を漏れなくリストアップし、実装の全体像を把握するため*

3. 🤖 **生成AI** - **コンポーネント分割案の生成**
    - 構成要素一覧をもとに、複数画面で共通利用できるコンポーネントを特定し、親子関係・再利用性を考慮した分割案を生成します。
    - *目的: コードの重複をなくし、保守性を高めるため*

4. 👤 **ユーザー** - **分割案のレビューと確定**
    - AIが提案した分割案を確認し、プロジェクトの実情（チーム規模・技術スタック等）に合わせて調整します。

5. 🤖 **生成AI** - **コンポーネント設計書の生成**
    - 確定した分割案をもとに、各コンポーネントの責務・受け取るprops・表示状態の定義をMarkdown形式で生成します。
    - *目的: 実装者が設計を読んですぐに実装に着手できるようにするため*
