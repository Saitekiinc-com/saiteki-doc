# タスクレビューフロー

## 【背景】なぜこのフローが必要なのか

issueを作成してタスクを開始する際、「このタスクでAI活用できる箇所があったのに、後から気づいた」という取りこぼしは珍しくありません。
このフローでは、GitHub Actionsがissue作成を検知し、ナレッジグラフを検索してAIがコメントを自動投稿することで、タスク開始時点でAI活用の観点を届けます。

### AI活用による課題解決のアプローチ

本カリキュラムでは、**「issue作成 → 自動ナレッジ検索 → AIコメント生成」** のパイプラインで、取りこぼしなくAI活用の観点を提供します。

1.  **キーワード自動検索**: issueのタイトル・本文からキーワードを抽出し、Neo4jのナレッジグラフを検索する。
2.  **ナレッジに基づくコメント**: 関連する「人の作業」「AIに任せられる作業」をGemini APIがコメントとして生成する。
3.  **タスクレビューとしての機能**: AIコメントがチームへの気づきを促し、AI活用の取りこぼしを防ぐ。

## フローの概要

GitHub Issueの作成をトリガーに、ナレッジグラフからAI活用ガイダンスを自動生成してコメントするフローです。

| Triggerのオブジェクト | 自動生成Output |
| :--- | :--- |
| **GitHub Issue**<br>タイトル・本文にタスクの概要を記述 | **AIコメント**<br>AIに任せられる作業 / 人が判断すべき作業の提示 |

### 協働フロー

1. 👤 **エンジニア (Human)** — **issueを作成**
    *   タスクの概要をissueのタイトルと本文に記述します。
    *   キーワードが多いほどナレッジ検索の精度が上がります（例：「テスト設計」「観点抽出」など）。

2. 🤖 **GitHub Actions** — **issueトリガーで自動起動**
    *   `issues.opened` イベントを検知してワークフローが起動します。
    *   `generate_ai_guidance.py` が実行され、issueタイトル・本文からキーワードを抽出します。

3. 🤖 **Neo4j + Gemini API** — **ナレッジ検索とコメント生成**
    *   キーワードでNeo4jの`HumanTask`・`AIGuidance`・`DomainKeyword`を検索します。
    *   関連ナレッジをGemini APIに渡し、issueに対するAI活用ガイダンスコメントを生成します。

4. 🤖 **GitHub Actions** — **issueにコメントを自動投稿**
    *   生成されたコメントがissueに自動投稿されます（30〜60秒）。
    *   コメントには「AIに任せられる作業」「人が判断する作業」が整理されて示されます。

5. 👤 **エンジニア (Human)** — **コメントを確認してタスクを進める**
    *   AI活用できる箇所を確認し、タスクの進め方を調整します。
    *   取りこぼしがあれば対応し、実際の作業に入ります。

```yaml
# .github/workflows/ai_guidance.yml
# issueが開かれたときに自動実行される設定
on:
  issues:
    types: [opened]
```
