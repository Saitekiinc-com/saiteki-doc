# フィードバックサイクル

## 【背景】なぜこのフローが必要なのか

AI活用の仮説は、実際のタスクを通じて検証・修正されることで精度が上がります。
「思ったよりAIが使えた」「ここは人が判断しないといけなかった」という気づきをNotionに追記し、ナレッジグラフを更新することで、次のタスクではより精度の高いAIコメントが届くようになります。

### AI活用による課題解決のアプローチ

本カリキュラムでは、**「実タスクの気づき → ナレッジ更新 → グラフ再生成」** のサイクルで、AIの活用精度を継続的に高めます。

1.  **仮説の検証サイクル**: タスクを通じた気づきをNotionに追記し、仮説を修正・強化する。
2.  **ナレッジの自動反映**: Markdownとグラフを再生成することで、最新の知見がすぐにシステムに反映される。
3.  **精度の継続的な向上**: フィードバックを重ねるほどAIコメントの質が向上し、チーム全体のAI活用力が高まる。

## フローの概要

タスク完了後の気づきをNotionに追記し、ナレッジグラフを更新して次のタスクに活かすフローです。

| 人のInput | 更新対象 |
| :--- | :--- |
| **Notionへの追記**<br>実タスクでの気づき・仮説の修正・新たなAI活用パターン | **GitHub Pages（Markdown）**<br>Neo4jナレッジグラフ（精度・フローの更新） |

### 協働フロー

1. 👤 **エンジニア (Human)** — **タスクを通じた気づきをNotionに追記**
    *   実際のタスクを経て、Notionの該当ページに気づきを追記します。
    *   以下の観点で記録します：
        *   AIに全量任せられた部分（仮説が正しかった箇所）
        *   思ったより人の判断が必要だった部分（修正すべき仮説）
        *   新たにAI活用できると感じた作業（新しい仮説）

2. 🤖 **自動パイプライン** — **Markdownを更新してGitHub Pagesに反映**
    *   `notion_to_markdown.py` を実行し、更新されたNotionをMarkdownに変換します。
    *   git pushするとGitHub Pagesが自動更新され、チームが最新のナレッジを参照できます。

3. 🤖 **自動パイプライン** — **ナレッジグラフを再生成**
    *   `notion_to_neo4j.py` を実行し、更新されたMarkdownをもとにNeo4jグラフを再構築します。
    *   次回のissue作成時から、より精度の高いAIコメントが生成されます。

```bash
# フィードバックをナレッジに反映する手順
export NOTION_TOKEN="your-notion-token"
export NEO4J_PASSWORD="your-neo4j-password"

# Step 1: 更新したNotionをMarkdownに変換してpush
python3 scripts/notion_to_markdown.py
git add docs/practices/ && git commit -m "docs: フィードバックをもとにナレッジ更新" && git push

# Step 2: ナレッジグラフを再生成（次のissueから精度が向上）
python3 scripts/notion_to_neo4j.py
```

> **継続的改善のポイント**: 一度に完璧なナレッジを求めず、小さな気づきを積み重ねることが重要です。タスクが終わるたびに1〜2行の追記をするだけでも、時間とともにナレッジは確実に成長します。
